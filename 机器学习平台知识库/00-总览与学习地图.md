## 0. 30 秒速通(TL;DR)

- **你的核心价值不是"会训模型",而是站在算法同学和老板中间,做三件事**:① **听得懂**算法黑话(slot、pass、穿越、GAUC……);② **接得住**算法需求,把它翻译成平台要落地的样本/特征能力;③ **翻译得了**,把技术问题翻译成 ROI、风险讲给老板听。
- **你的主场是两块**:**样本数据管理**(日志怎么拼成样本、标签怎么打、怎么采样、离在线怎么一致)和**特征数据管理**(特征怎么生产、怎么存、怎么保证不穿越、怎么离在线一致)。算法和模型只需要"懂到能对话"。
- **这个知识库共 16 篇正文 + 本篇总览**,分 6 个主题文件夹:业务场景、训练名词、特征管理、样本管理、平台架构、模型演进与话术。
- **学习顺序建议**:先读「02 训练名词」打通黑话 → 再深耕你的主场「03 特征 + 04 样本」→ 然后「01 业务 + 05 链路」把上下游串起来 → 最后「06 模型演进 + 话术」补充弹药。
- **一句话速记**:**懂业务才知道为什么,懂算法才听得懂,懂数据才接得住,懂表达才出得彩。**

> 💡 这个知识库刻意**不教你训模型**。市面上讲模型的资料铺天盖地,但没人教"平台研发该懂到什么程度"。这里给你的是**恰到好处的深度**——深到能在评审里压住场子,又不浪费时间在你用不上的调参细节上。

## 1. 你是谁:平台研发在搜广推团队里的真实定位

很多平台研发对自己的定位有误解,以为自己是"给算法同学搬数据的工具人"。**这是最危险的认知**——一旦你这么想,你就会被当成工具人对待。

正确的定位是:**你是算法效果的"供给侧基础设施"负责人**。算法同学的模型再牛,吃进去的是你供给的样本和特征。**料不对,神仙难救。**

```
        ┌─────────────────────────────────────────────────────┐
        │                  搜广推算法团队                       │
        │                                                       │
        │   算法/建模同学          训练/工程同学                 │
        │   (设计模型结构)        (把模型训起来、上线)          │
        │        │                      │                       │
        │        └──────────┬───────────┘                       │
        │                   │ 都依赖                             │
        │                   ▼                                   │
        │        ┌──────────────────────┐                       │
        │        │   样本 + 特征 供给     │  ← 这就是你           │
        │        │   (平台研发的主场)    │                       │
        │        └──────────────────────┘                       │
        └─────────────────────────────────────────────────────┘
```

**三种角色,三种语言**:

| 角色 | 关心什么 | 常说的话 | 你要怎么接 |
|---|---|---|---|
| 算法/建模同学 | 模型结构、特征质量、AUC/GAUC | "这个特征穿越了""加个序列特征""slot 不够用" | 翻译成样本/特征链路的改造 |
| 训练/工程同学 | 训练吞吐、稳定性、上线时效 | "pass 卡住了""样本读不动""embedding 太大 OOM" | 翻译成数据供给的性能/格式问题 |
| 老板 | 业务收益、成本、风险 | "这个季度涨了几个点?""这事值不值得投入?" | 翻译成 ROI、事故风险、人力成本 |

> 🎓 **老师视角**:你能不能"脱颖而出",取决于你能否在这三种语言之间**自由切换**。算法同学说"slot 穿越",你既能听懂(特征时间错位),又能立刻判断这是样本拼接链路的时间对齐问题(接得住),还能在老板问起时说"这个 bug 会让我们离线虚高 2 个点、上线掉 3 个点,等于白干一个迭代"(翻译得了)。**这三层能力,就是这个知识库要给你的。**

## 2. 知识坐标系:业务 / 算法 / 数据 / 平台 四象限

把你要懂的知识放进一张图,你就知道自己缺哪块:

```
              懂"为什么"                          懂"怎么做"
        ┌──────────────────────┬──────────────────────┐
  上    │   ① 业务场景           │   ② 算法/模型         │
  游    │   (搜广推三业务、       │   (排序模型、训练范式、 │
  视    │    指标、漏斗、竞价)    │    评估指标、训练名词)  │
  角    │   → 文档 01            │   → 文档 02 / 06       │
        ├──────────────────────┼──────────────────────┤
  你    │   ③ 数据(你的主场)    │   ④ 平台/系统          │
  的    │   (样本管理、特征管理、 │   (Feature Store、     │
  主    │    标签、采样、一致性)  │    样本平台、训练链路、 │
  场    │   → 文档 03 / 04       │    数据湖底座)         │
        │                       │   → 文档 05            │
        └──────────────────────┴──────────────────────┘
              你要"精通"③④                你要"听懂"①②
```

**结论很清晰**:

- **③ 数据 + ④ 平台**:你的主场,要**精通**——能设计、能落地、能在评审里讲清架构与权衡。对应文档 03(特征)、04(样本)、05(平台链路)。
- **① 业务 + ② 算法**:你的"必修外语",要**听得懂、对话得起来**——不需要会推公式、会调参,但要知道每个概念背后对你数据供给的要求。对应文档 01(业务)、02(训练名词)、06(模型演进)。

## 5. 三条学习路径:按你的目标选路

**路径 A · 救急(明天就要开会,只有 2 小时)**
> `02a pass·slot` → `06b 话术与高频问答` → 本篇第 3 节那张全景图。
> 目标:不在评审里露怯,能接住黑话,能画出闭环图。

**路径 B · 立本(系统补齐主场,1~2 周)**
> `02a/02b/02c`(打通黑话)→ `04-00 一致性` → `04a/04b/04c`(吃透样本)→ `03a/03b/03c`(吃透特征)→ `05a/05b`(串成系统)。
> 目标:样本和特征两块成为团队里说得最清楚的人。

**路径 C · 拔高(想主导平台架构,1 个月)**
> 路径 B 全部 + `01a/01b`(业务为什么)+ `06a`(模型演进,理解算法同学的诉求来源)。
> 目标:能独立设计样本/特征平台、能在跨团队评审里定方案、能把平台价值讲到老板心里。

> 🎓 **老师视角**:大多数人卡在"只走了路径 A 就以为自己懂了"。真正拉开差距的是路径 B 的 `04a/04b/04c` 和 `03b/03c`——**这些是你的专业护城河**,是算法同学不会、但极度依赖的东西。把这几篇讲透,你在团队里就从"可替代"变成"绕不开"。

## 7. 平台研发的"能力分层":从工具人到不可替代

```

   ▲      (例:算法要实时样本，你能权衡 Flink 拼接 vs 快照回流，给出方案与成本)
   │
 Level 3  接得住:听懂算法诉求，独立翻译成样本/特征链路改造并落地
   │      (例:算法说"加行为序列特征"，你知道要改样本 schema、特征生产、在线 KV)
   │
 Level 2  听得懂:搜广推黑话无障碍，能参与技术讨论不掉线
   │      (例:听到"slot 穿越""GAUC 没涨"知道在说什么)
   │
 Level 1  搬得动:能按需求把数据跑出来，但不懂为什么
          (工具人区，最容易被替代)
```

> 🎓 **老师视角**:这个知识库的目标是把你从 Level 1/2 推到 **Level 3/4**。区别在哪?Level 2 是"听懂别人说",Level 3/4 是"我能定义问题、给出方案"。后者的标志是:**你开始反过来教算法同学"你这个特征会穿越,得这么取数"**——当你能纠正算法同学的数据用法时,你就不可替代了。

## 9. 自测三问

1. **用一句话说清你作为平台研发在搜广推团队的价值定位。**
   > 参考:我是算法效果的"供给侧基础设施"负责人——模型吃进去的样本和特征都由我供给,料不对模型再牛也没用;我的价值在于"听得懂算法、接得住需求、向上翻译得了 ROI"。

2. **不看文档,默画第 3 节那张"一条样本到一次收入"的闭环图,标出 ①~⑤ 五个环节。**
   > 参考:①在线打分打特征快照 → ②用户行为产生曝光/点击/转化日志 → ③离在线一致地拼样本 → ④样本流/批喂训练 → ⑤产出 base+delta 模型灰度上线监控回流。底座是数据湖。

3. **你的"专业护城河"是四象限里的哪两块?为什么不是另外两块?**
   > 参考:③数据(样本/特征管理)+ ④平台架构,因为这是算法同学不会、却极度依赖的能力;①业务②算法只需"听得懂",会公式调参不是平台研发的核心竞争力。
