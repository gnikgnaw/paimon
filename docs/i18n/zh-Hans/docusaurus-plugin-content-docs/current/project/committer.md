# Committer {#committer}

## 成为 Committer {#become-a-committer}

#### 如何成为 Committer {#how-to-become-a-committer}

成为 committer 并没有严格的流程。新 committer 的候选人通常是活跃的贡献者和社区成员。候选人由现任 committer 或 PMC 成员提名，并由 PMC 投票表决。

如果你希望成为 committer，你应当积极参与社区，并以上述任意一种方式开始为 Apache Paimon 做贡献。你也可以与其他 committer 交流，征求他们的建议和指导。

- 社区贡献包括：在邮件列表上帮助回答用户问题、验证发布候选版本、做技术分享、组织社区活动，以及其他形式的布道与社区建设。「Apache 之道（Apache Way）」高度重视项目社区，即便没有任何代码贡献，committer 也可以因杰出的社区贡献而获得认可。

- 代码／技术贡献包括：提交 pull request（补丁）、参与设计讨论、做代码评审、测试，以及在发现和修复 bug 方面提供的其他帮助。尤其是富有建设性且高质量的设计讨论，以及帮助其他贡献者，都是有力的信号。

#### 发现有潜力的候选人 {#identify-promising-candidates}

虽然前面几点提供了发现有潜力候选人的途径，但以下几点是任何 committer 候选人都「必须具备」的：

- 具有社区意识：候选人理解社区治理中的精英主义（meritocratic）原则。他们并不总是追求尽可能多的个人贡献，而是会在合理之处帮助并赋能他人。

- 我们相信 committer 候选人会负责任地使用其对仓库的写权限，如有疑虑则保守行事。重要的是，committer 要清楚自己知道什么、不知道什么。如有疑虑，committer 应当寻求第二个人的把关，而不是直接向自己并不十分熟悉的部分提交代码。

- 他们已经表现出对其他社区成员的尊重，并在讨论中保持建设性。
