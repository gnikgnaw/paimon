# Hive

本文档是在 Hive 中使用 Paimon 的指南。

## 版本 {#version}

Paimon 当前支持 Hive 3.1、2.3、2.2、2.1 和 2.1-cdh-6.3。

## 执行引擎 {#execution-engine}

Paimon 当前在 Hive 读取时支持 MR 和 Tez 执行引擎，在 Hive 写入时支持 MR 执行引擎。
注意：如果你使用 beeline，请重启 hive 集群。

你也可以从源代码手动构建 bundled jar。

要从源代码构建，请[克隆 git 仓库](@@GITHUB_REPO@@)。

使用以下命令构建 bundled jar。
`mvn clean install -DskipTests`

你可以在 `./paimon-hive/paimon-hive-connector-<hive-version>/target/paimon-hive-connector-<hive-version>-@@VERSION@@.jar` 中找到 Hive connector jar。

有几种方式可以将该 jar 添加到 Hive。

* 你可以在 Hive 的根目录下创建一个 `auxlib` 文件夹，并将 `paimon-hive-connector-@@VERSION@@.jar` 复制到 `auxlib` 中。
* 你也可以将该 jar 复制到 Hive 可访问的路径下，然后使用 `add jar /path/to/paimon-hive-connector-@@VERSION@@.jar` 在 Hive 中启用 paimon 支持。注意，不推荐使用此方法。如果你使用 MR 执行引擎并运行 join 语句，可能会遇到异常 `org.apache.hive.com.esotericsoftware.kryo.kryoexception: unable to find class`。

注意：

* 如果你使用 HDFS：
  * 确保已设置环境变量 `HADOOP_HOME` 或 `HADOOP_CONF_DIR`。
  * 你可以设置 `paimon.hadoop-load-default-config` =`false` 来禁止从 `core-default.xml`、`hdfs-default.xml` 加载默认值，这可能会导致 split 的尺寸更小。
* 在使用 hive cbo 时，可能会导致部分查询结果不正确，例如查询带 `not null` 谓词的 `struct` 类型时，你可以通过 `set hive.cbo.enable=false;` 命令禁用 cbo。

## Hive SQL：访问 Hive metastore 中已有的 Paimon 表 {#hive-sql-access-paimon-tables-already-in-hive-metastore}

在 Hive CLI 中运行以下 Hive SQL 来访问已创建的表。

```sql
-- Assume that paimon-hive-connector-<hive-version>-@@VERSION@@.jar is already in auxlib directory.
-- List tables in Hive
-- (you might need to switch to "default" database if you're not there by default)

SHOW TABLES;

/*
OK
test_table
*/

-- Read records from test_table

SELECT a, b FROM test_table ORDER BY a;

/*
OK
1	Table
2	Store
*/

-- Insert records into test table
-- Limitations:
-- Only support INSERT INTO, not support INSERT OVERWRITE
-- It is recommended to write to a non primary key table
-- Writing to a primary key table may result in a large number of small files

INSERT INTO test_table VALUES (3, 'Paimon');

SELECT a, b FROM test_table ORDER BY a;

/*
OK
1	Table
2	Store
3	Paimon
*/

-- time travel

SET paimon.scan.snapshot-id=1;
SELECT a, b FROM test_table ORDER BY a;
/*
OK
1	Table
2	Store
3	Paimon
*/
SET paimon.scan.snapshot-id=null;

```

## Hive SQL：创建新的 Paimon 表 {#hive-sql-create-new-paimon-tables}

你可以在 Hive 中创建新的 paimon 表。在 Hive CLI 中运行以下 Hive SQL。

```sql
-- Assume that paimon-hive-connector-@@VERSION@@.jar is already in auxlib directory.
-- Let's create a new paimon table.

SET hive.metastore.warehouse.dir=warehouse_path;

CREATE TABLE hive_test_table(
    a INT COMMENT 'The a field',
    b STRING COMMENT 'The b field'
)
STORED BY 'org.apache.paimon.hive.PaimonStorageHandler';
```

## Hive SQL：通过外部表访问 Paimon 表 {#hive-sql-access-paimon-tables-by-external-table}

要访问已有的 paimon 表，你也可以将它们注册为 Hive 中的外部表。在 Hive CLI 中运行以下 Hive SQL。

```sql
-- Assume that paimon-hive-connector-@@VERSION@@.jar is already in auxlib directory.
-- Let's use the test_table created in the above section.
-- To create an external table, you don't need to specify any column or table properties.
-- Pointing the location to the path of table is enough.

CREATE EXTERNAL TABLE external_test_table
STORED BY 'org.apache.paimon.hive.PaimonStorageHandler'
LOCATION '/path/to/table/store/warehouse/default.db/test_table';
    
-- In addition to the way setting location above, you can also place the location setting in TBProperties
-- to avoid Hive accessing Paimon's location through its own file system when creating tables.
-- This method is effective in scenarios using Object storage,such as s3.

CREATE EXTERNAL TABLE external_test_table
STORED BY 'org.apache.paimon.hive.PaimonStorageHandler'
TBLPROPERTIES (
 'paimon_location' ='s3://xxxxx/path/to/table/store/warehouse/default.db/test_table'
);

-- Read records from external_test_table

SELECT a, b FROM external_test_table ORDER BY a;

/*
OK
1	Table
2	Store
*/

-- Insert records into test table

INSERT INTO external_test_table VALUES (3, 'Paimon');

SELECT a, b FROM external_test_table ORDER BY a;

/*
OK
1	Table
2	Store
3	Paimon
*/

```
