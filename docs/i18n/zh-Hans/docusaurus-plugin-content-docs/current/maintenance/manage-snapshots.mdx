# 管理快照 {#manage-snapshots}

本节将介绍与快照相关的管理与行为。

## 快照过期 {#expire-snapshots}

Paimon 写入端每次提交都会生成一个或两个[快照（snapshot）](../concepts/basic-concepts#snapshot)。每个快照可能会新增一些数据文件，或将某些旧数据文件标记为已删除。然而，被标记的数据文件并不会真正删除，因为 Paimon 还支持时间旅行（time travel）到更早的快照。只有当快照过期时，它们才会被删除。

目前，过期由 Paimon 写入端在提交新变更时自动执行。通过让旧快照过期，可以删除不再使用的旧数据文件和元数据文件，从而释放磁盘空间。

请注意，保留时间过短或保留数量过小可能导致：

- 批式查询无法找到文件。例如，表相对较大，批式查询需要 10 分钟才能读取完成，但 10 分钟前的快照已过期，此时批式查询将读取到一个已删除的快照。
- 表文件上的流式读取作业无法重新启动。
  当作业重启时，它记录的快照可能已经过期。（你可以使用
  [Consumer Id](../flink/consumer-id) 在快照过期保留时间较短的情况下保护流式读取）。

默认情况下，Paimon 会同步删除过期的快照。当需要删除的文件过多时，可能无法快速删除，并对上游算子产生反压。为避免这种情况，用户可以通过将 `snapshot.expire.execution-mode` 设置为 `async` 来使用异步过期模式。但是，如果你的作业以批式模式运行，则不建议使用异步过期模式，因为过期任务可能无法成功完成。

## 手动让快照过期 {#manually-expire-snapshot}
手动让某张表的快照过期
<Tabs groupId="manually-expire-snapshot">

<TabItem value="flink-sql" label="Flink SQL">

运行以下命令：

```sql
-- for Flink 1.18
CALL sys.expire_snapshots('database_name.table_name', 2)
-- for Flink 1.19 and later
CALL sys.expire_snapshots(`table` => 'database_name.table_name', retain_max => 2)
CALL sys.expire_snapshots(`table` => 'database_name.table_name', older_than => '2024-01-01 12:00:00')
CALL sys.expire_snapshots(`table` => 'database_name.table_name', older_than => '2024-01-01 12:00:00', retain_min => 10)
CALL sys.expire_snapshots(`table` => 'database_name.table_name', older_than => '2024-01-01 12:00:00', max_deletes => 10, options => 'snapshot.expire.limit=1')
```

</TabItem>

<TabItem value="flink-action" label="Flink Action">

运行以下命令：

```bash
<FLINK_HOME>/bin/flink run \
    /path/to/paimon-flink-action-@@VERSION@@.jar \
    expire_snapshots \
    --warehouse <warehouse-path> \
    --identifier <identifier> \
    --older_than <timestamp> \
    --version <snapshot-id> \
    --max_deletes <max-deletes> \
    --retain_max <retain-max> \
    --retain_min <retain-min> \
    [--catalog_conf <paimon-catalog-conf> [--catalog_conf <paimon-catalog-conf> ...]]
```

</TabItem>

<TabItem value="spark" label="Spark">

运行以下 SQL：

```sql
CALL sys.expire_snapshots(table => 'database_name.table_name', retain_max => 10, options => 'snapshot.expire.limit=1');
```

</TabItem>

</Tabs>

## 回滚到快照 {#rollback-to-snapshot}

将表回滚到指定的快照 ID。

<Tabs groupId="rollback-to">

<TabItem value="flink-sql" label="Flink SQL">

运行以下命令：

```sql
CALL sys.rollback_to(`table` => 'database_name.table_name', snapshot_id => <snasphot-id>);
```

</TabItem>

<TabItem value="flink-action" label="Flink Action">

运行以下命令：

```bash
<FLINK_HOME>/bin/flink run \
    /path/to/paimon-flink-action-@@VERSION@@.jar \
    rollback_to \
    --warehouse <warehouse-path> \
    --database <database-name> \
    --table <table-name> \
    --version <snapshot-id> \
    [--catalog_conf <paimon-catalog-conf> [--catalog_conf <paimon-catalog-conf> ...]]
```

</TabItem>

<TabItem value="java-api" label="Java API">

```java
import org.apache.paimon.table.Table;

public class RollbackTo {

    public static void main(String[] args) {
        // before rollback:
        // snapshot-3
        // snapshot-4
        // snapshot-5
        // snapshot-6
        // snapshot-7
      
        table.rollbackTo(5);
        
        // after rollback:
        // snapshot-3
        // snapshot-4
        // snapshot-5
    }
}
```

</TabItem>

<TabItem value="spark" label="Spark">

运行以下 SQL：

```sql
CALL sys.rollback(table => 'database_name.table_name', snapshot => snasphot_id);
```

</TabItem>

</Tabs>

## 移除孤儿文件 {#remove-orphan-files}

Paimon 文件只有在快照过期时才会被物理删除。然而，在删除文件时可能发生某些意外错误，从而导致存在未被任何 Paimon 快照使用的文件（即所谓的“孤儿文件”）。你可以提交一个 `remove_orphan_files` 作业来清理它们：

<Tabs groupId="remove-orphan-files">

<TabItem value="spark-sql-flink-sql" label="Spark SQL/Flink SQL">

```sql
CALL sys.remove_orphan_files(`table` => 'my_db.my_table', [older_than => '2023-10-31 12:00:00'])

CALL sys.remove_orphan_files(`table` => 'my_db.*', [older_than => '2023-10-31 12:00:00'])
```

</TabItem>

<TabItem value="flink-action" label="Flink Action">

```bash
<FLINK_HOME>/bin/flink run \
    /path/to/paimon-flink-action-@@VERSION@@.jar \
    remove_orphan_files \
    --warehouse <warehouse-path> \
    --database <database-name> \
    --table <table-name> \
    [--older_than <timestamp>] \
    [--dry_run <false/true>] \
    [--parallelism <parallelism>]
```

为避免删除其他写入作业新增的文件，该 Action 默认只删除超过 1 天的孤儿文件。该时间间隔可以通过 `--older_than` 修改。例如：

```bash
<FLINK_HOME>/bin/flink run \
    /path/to/paimon-flink-action-@@VERSION@@.jar \
    remove_orphan_files \
    --warehouse <warehouse-path> \
    --database <database-name> \
    --table T \
    --older_than '2023-10-31 12:00:00'
```

table 可以设为 `*`，以清理数据库中的所有表。

</TabItem>

</Tabs>
