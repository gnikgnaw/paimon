# 文件系统 {#filesystems}

Apache Paimon 使用与 Apache Flink 相同的可插拔文件系统。如果使用 Flink 作为计算引擎，用户可以遵循
[标准插件机制](https://nightlies.apache.org/flink/flink-docs-stable/docs/deployment/filesystems/plugins/)
来配置插件结构。然而，对于 Spark 或 Hive 等其他引擎，Flink 提供的 opt jar 可能会产生冲突，无法直接使用。
用户修复类冲突并不方便，因此 Paimon 提供了自包含、引擎统一的 FileSystem 可插拔 jar，供用户从 Spark/Hive 端查询表。

## 支持的文件系统 {#supported-filesystems}

| 文件系统                       | URI Scheme | 可插拔 | 说明                                                                   |
|:-----------------------------|:-----------|-----------|:-----------------------------------------------------------------------|
| 本地文件系统                   | file://    | N         | 内置支持                                                                |
| HDFS                         | hdfs://    | N         | 内置支持，请确保集群处于 hadoop 环境中                                     |
| Aliyun OSS                   | oss://     | Y         |                                                                        |
| S3                           | s3://      | Y         |                                                                        |
| Tencent Cloud Object Storage | cosn://    | Y         |                                                                        |
| Microsoft Azure Storage      | abfs://    | Y         |                                                                        |
| Huawei OBS                   | obs://     | Y         |                                                                        |
| Google Cloud Storage         | gs://      | Y         |                                                                        |

## 依赖 {#dependency}

我们建议你直接下载 jar：[下载链接](../project/download#filesystem-jars)。

你也可以从源代码手动构建 bundled jar。

要从源代码构建，请[克隆 git 仓库](@@GITHUB_REPO@@)。

使用以下命令构建 shaded jar。

```bash
mvn clean install -DskipTests
```

你可以在以下路径找到 shaded jar：
`./paimon-filesystems/paimon-${fs}/target/paimon-${fs}-@@VERSION@@.jar`。

## HDFS {#hdfs}

访问 HDFS 不需要任何额外的依赖，因为你已经处理好了 Hadoop 依赖。

### HDFS 配置 {#hdfs-configuration}

对于 HDFS，最重要的是能够读取你的 HDFS 配置。

<Tabs groupId="hdfs-conf">

<TabItem value="flink" label="Flink">

如果你处于 hadoop 环境中，可能无需做任何事情。否则，请选择以下方式之一来配置你的 HDFS：

1. 设置环境变量 `HADOOP_HOME` 或 `HADOOP_CONF_DIR`。
2. 在 paimon catalog 中配置 `'hadoop-conf-dir'`。
3. 在 paimon catalog 中通过前缀 `'hadoop.'` 配置 Hadoop 选项。

推荐使用第一种方式。

如果你不想包含该环境变量的值，可以将 `hadoop-conf-loader` 配置为 `option`。

</TabItem>

<TabItem value="hive-spark" label="Hive/Spark">

HDFS 配置可直接通过计算集群获取，详见 Hive 和 Spark 的集群配置。

</TabItem>

</Tabs>

### Hadoop 兼容文件系统（HCFS） {#hadoop-compatible-file-systems-hcfs}

当 Hadoop 库位于 classpath 上时，所有 Hadoop 文件系统都会自动可用。

通过这种方式，Paimon 无缝支持所有实现了 `org.apache.hadoop.fs.FileSystem`
接口的 Hadoop 文件系统，以及所有 Hadoop 兼容文件系统（HCFS）。

- HDFS
- Alluxio（具体配置见下文）
- XtreemFS
- …

Hadoop 配置必须在 `core-site.xml` 文件中为所需的文件系统实现包含一个条目。

要支持 Alluxio，请将以下条目添加到 core-site.xml 文件中：

```xml
<property>
  <name>fs.alluxio.impl</name>
  <value>alluxio.hadoop.FileSystem</value>
</property>
```

### Kerberos {#kerberos}

<Tabs groupId="kerberos">

<TabItem value="flink" label="Flink">

在你的 catalog 配置中配置以下选项：

- security.kerberos.login.keytab：包含用户凭据的 Kerberos keytab 文件的绝对路径。
  请确保它已复制到每台机器上。
- security.kerberos.login.principal：与 keytab 关联的 Kerberos principal 名称。

并在程序的 java property 中配置以下选项：

- java.security.krb5.conf：Kerberos 配置文件的绝对路径。
  请确保它已复制到每台机器上。

</TabItem>

<TabItem value="spark" label="Spark">

建议使用 [Spark Kerberos Keytab](https://spark.apache.org/docs/latest/security.html#using-a-keytab)。

</TabItem>

<TabItem value="hive" label="Hive">

一种直观的方式是配置 Hive 的 kerberos 认证。

</TabItem>

<TabItem value="trino-javaapi" label="Trino/JavaAPI">

在你的 catalog 配置中配置以下三个选项：

- security.kerberos.login.keytab：包含用户凭据的 Kerberos keytab 文件的绝对路径。
  请确保它已复制到每台机器上。
- security.kerberos.login.principal：与 keytab 关联的 Kerberos principal 名称。
- security.kerberos.login.use-ticket-cache：True 或 false，表示是否从你的 Kerberos ticket cache 中读取。

对于 JavaAPI：
```
SecurityContext.install(catalogOptions);
```

</TabItem>

</Tabs>

### HDFS HA {#hdfs-ha}

确保 `hdfs-site.xml` 和 `core-site.xml` 包含必要的 [HA 配置](https://hadoop.apache.org/docs/stable/hadoop-project-dist/hadoop-hdfs/HDFSHighAvailabilityWithNFS.html)。

### HDFS ViewFS {#hdfs-viewfs}

确保 `hdfs-site.xml` 和 `core-site.xml` 包含必要的 [ViewFs 配置](https://hadoop.apache.org/docs/stable/hadoop-project-dist/hadoop-hdfs/ViewFs.html)。

## OSS {#oss}

<Stable>



下载 [paimon-oss-@@VERSION@@.jar](https://repo.maven.apache.org/maven2/org/apache/paimon/paimon-oss/@@VERSION@@/paimon-oss-@@VERSION@@.jar)。



</Stable>

<Unstable>

</Unstable>

<Tabs groupId="oss">

<TabItem value="flink" label="Flink">

:::info

如果你已经[通过 Flink 配置了 oss 访问](https://nightlies.apache.org/flink/flink-docs-stable/docs/deployment/filesystems/oss/)（通过 Flink FileSystem），
那么可以跳过以下配置。

:::

将 `paimon-oss-@@VERSION@@.jar` 放入你 Flink home 的 `lib` 目录，并创建 catalog：

```sql
CREATE CATALOG my_catalog WITH (
    'type' = 'paimon',
    'warehouse' = 'oss://<bucket>/<path>',
    'fs.oss.endpoint' = 'oss-cn-hangzhou.aliyuncs.com',
    'fs.oss.accessKeyId' = 'xxx',
    'fs.oss.accessKeySecret' = 'yyy'
);
```

</TabItem>

<TabItem value="spark" label="Spark">

:::info

如果你已经通过 Spark 配置了 oss 访问（通过 Hadoop FileSystem），那么可以跳过以下配置。

:::

将 `paimon-oss-@@VERSION@@.jar` 与 `paimon-spark-@@VERSION@@.jar` 一起放到 Spark 的 jars 目录下，并按如下方式启动

```shell
spark-sql \
  --conf spark.sql.extensions=org.apache.paimon.spark.extensions.PaimonSparkSessionExtensions \
  --conf spark.sql.catalog.paimon=org.apache.paimon.spark.SparkCatalog \
  --conf spark.sql.catalog.paimon.warehouse=oss://<bucket>/<path> \
  --conf spark.sql.catalog.paimon.fs.oss.endpoint=oss-cn-hangzhou.aliyuncs.com \
  --conf spark.sql.catalog.paimon.fs.oss.accessKeyId=xxx \
  --conf spark.sql.catalog.paimon.fs.oss.accessKeySecret=yyy
```

</TabItem>

<TabItem value="hive" label="Hive">

:::info

如果你已经通过 Hive 配置了 oss 访问（通过 Hadoop FileSystem），那么可以跳过以下配置。

:::

注意：你需要确保 Hive metastore 能够访问 `oss`。

将 `paimon-oss-@@VERSION@@.jar` 与 `paimon-hive-connector-@@VERSION@@.jar` 一起放到 Hive 的 auxlib 目录下，并按如下方式启动

```sql
SET paimon.fs.oss.endpoint=oss-cn-hangzhou.aliyuncs.com;
SET paimon.fs.oss.accessKeyId=xxx;
SET paimon.fs.oss.accessKeySecret=yyy;
```

然后从 hive metastore 读取表，该表可以由 Flink 或 Spark 创建，参见 [使用 Hive Metastore 的 Catalog](../flink/sql-ddl)
```sql
SELECT * FROM test_table;
SELECT COUNT(1) FROM test_table;
```

</TabItem>

<TabItem value="trino" label="Trino">

从 0.8 版本开始，paimon-trino 使用 trino filesystem 作为基础的文件读写系统。我们强烈建议你在 trino 中使用 jindo-sdk。

你可以在此处找到 [如何在 trino 上配置 jindo sdk](https://github.com/aliyun/alibabacloud-jindodata/blob/master/docs/user/4.x/4.6.x/4.6.12/oss/presto/jindosdk_on_presto)。
请注意：
* 当你解压插件 jar 并找到放置位置时，使用 paimon 替换 hive-hadoop2。
* 你可以在配置项 [hive.config.resources](https://trino.io/docs/current/connector/hive.html#hdfs-configuration) 中的 `paimon.properties` 里指定 `core-site.xml`。
* Presto 与 Jindo 使用相同的配置方法。

</TabItem>

</Tabs>

如果你的环境有 jindo sdk 依赖，你可以使用 Jindo Fs 来连接 OSS。Jindo 具有更好的读写效率。

### S3 兼容对象存储 {#s3-compliant-object-stores}

S3 Filesystem 同样支持使用 S3 兼容的对象存储，例如 MinIO、腾讯的 COS 和 IBM 的 Cloud Object
Storage。只需将你的 endpoint 配置为对象存储服务的提供方即可。

```yaml
s3.endpoint: your-endpoint-hostname
```

### 配置 Path Style 访问 {#configure-path-style-access}

某些 S3 兼容的对象存储可能默认未启用 virtual host style 寻址，例如出于测试目的使用 Standalone MinIO 时。
在这种情况下，你需要提供启用 path style 访问的属性。

```yaml
s3.path.style.access: true
```

### S3A 性能 {#s3a-performance}

为 `S3AFileSystem` [调优性能](https://hadoop.apache.org/docs/stable/hadoop-aws/tools/hadoop-aws/performance.html)。

如果你遇到以下异常：
```shell
Caused by: org.apache.http.conn.ConnectionPoolTimeoutException: Timeout waiting for connection from pool.
```
请尝试在 catalog 配置项中配置：`fs.s3a.connection.maximum=1000`。
